# 双臂之间121个位置抓取与IK阶段分析（2026-07-19）

## 实验范围

G1 初始左、右 Inspire base 世界坐标分别约为 `(-0.302, 0.234, 0.844) m` 和
`(0.003, 0.234, 0.844) m`。本实验用11×11网格覆盖：

- X：-0.30 到 0.00 m，间距30 mm，覆盖左右手之间的横向范围；
- Y：0.26 到0.46 m，间距20 mm；
- Z：0.86 m；
- 每个位置重新采集 RGB-D，HUG 生成8候选，使用 xr_teleoperate G1_29_ArmIK 执行。

运行命令：

```bash
PYTHONPATH="$PWD/src:$PYTHONPATH" conda run --no-capture-output -n unitree_sim_env \
python -m lab_g1_collect.sim_smoke --device cpu --headless --steps 80000 --episodes 121 \
  --auto-collect --object-shape cylinder --object-fixed-xyz -0.150 0.360 0.860 \
  --object-grid-xy 11 --object-grid-step-xy 0.030 0.020 --seed 20260805 \
  --hug-candidates 8 --hug-sampling-steps 5 --arm-ik xr_teleoperate \
  --output outputs/xr_ik_between_arms_11x11
```

## 分类定义

- `P`：pre-grasp 终点在额外100控制步内仍未进入40 mm位置容差；直线接近尚未开始。
- `A`：pre-grasp 已到达，但 pre-grasp→grasp 直线段横向偏差超过40 mm，且最终 grasp 未到达。
- `G`：pre-grasp 与 grasp 都到达，完整执行闭手和抬升。
- `L`：在 G 的基础上，末段物体最低抬升不少于30 mm，即实际抓起。

## 两个实际抓起位置

### 网格 (-0.30, 0.28, 0.86) m

物体因靠近左手发生初始接触，稳定后的实测位置为约 `(-0.295, 0.268, 0.855) m`，所以
该点不应视为干净的固定位置证据。HUG grasp 为 `(-0.305, 0.154, 0.898) m`，pre-grasp
为 `(-0.314, 0.061, 0.934) m`，腕到物体距离122.8 mm。物体抬升46.5 mm，但线速度
0.399 m/s、角速度11.65 rad/s，严格质量失败。

### 网格 (-0.21, 0.28, 0.86) m

该点稳定且是目前最可靠的成功位置。HUG grasp 为 `(-0.158, 0.200, 0.983) m`，
pre-grasp 为 `(-0.125, 0.149, 1.062) m`，腕到物体距离156.0 mm。物体抬升54.4 mm，
指尖距离和线速度均通过，只有最大角速度5.02 rad/s超过1 rad/s严格阈值。

## 三阶段具体结论

### 1. pre-grasp 目标问题

105个失败发生在直线接近开始前。最终位置误差的五数概括为：最小40.8 mm、25分位
53.9 mm、中位81.4 mm、75分位119.9 mm、最大295.3 mm。

完整 IK 与 pre-grasp 失败组的“选中腕到物体中心距离”均值分别为178.3 mm和178.7 mm，
几乎完全相同，所以仅看这个标量无法判断可执行性。更明显的差异是完整6D姿态及其外推
目标：完整 IK 组 pre-grasp 平均坐标约为 `(-0.033, 0.133, 0.979) m`，失败组约为
`(-0.080, 0.200, 1.049) m`；失败组平均高70 mm，并且腕旋转分布明显不同。

### 2. 直线接近问题

只有2组属于此类：

- 网格 `(-0.27, 0.30)`：在直线路径86.6%处首次超过40 mm跟踪误差，最大横向偏差
  42.7 mm，最终 grasp 误差42.1 mm；
- 网格 `(-0.21, 0.34)`：在路径49.7%处首次超过40 mm，最大横向偏差58.0 mm，最终
  grasp 误差55.5 mm。

两条轨迹最大倒退进度分别只有0.0012和0.0029，说明不是控制器明显先前进再后退；主要
是末端逐渐偏离笛卡尔直线。

### 3. grasp 终点问题

没有观察到“直线横向误差不超过40 mm，但只在 grasp 终点超时”的样本。当前 grasp
超时都是直线 approach 已经偏离后的结果，因此不能归结为 grasp 点单独不可达。

## 为什么位置在工作空间内，IK仍然失败

这里的“到不了”是 Isaac 中实测末端没有跟踪到目标，不等价于数学上证明无 IK 解：

1. HUG 给的是完整6D腕位姿，不只是三维位置。位置在右臂几何工作空间内，不代表同一
   位置配合任意腕旋转都能在7关节限位内实现。
2. pre-grasp 不是 HUG 直接预测点，而是从 HUG grasp 沿“物体中心→腕”的径向再外推
   100 mm，并沿插值同时把腕旋转到 HUG 姿态。这个构造可能比 grasp 本身更高、更靠后，
   因而首先失败。
3. xr_teleoperate IK 在其 Pinocchio/URDF模型中求解；命令进入 Isaac 后会按当前 USD 的
   0.9软限位裁剪，再由 PhysX关节控制器跟踪。模型、末端 frame correction、软限位和
   动态跟踪误差都会累积。
4. 关节限位不是唯一原因。按 episode 关节轨迹回算，pre-grasp失败组有76/105进入
   0.08 rad限位警戒，而完整IK组也有11/14进入警戒；完整IK组甚至有7/14到过10 mrad
   以内。红色限位提示只能说明风险，不能单独证明不可达。

因此下一步最有效的修改不是继续仅按腕到物体距离挑 HUG 候选，而是对8个候选分别构造
pre-grasp，并用在线 G1 IK 的目标残差、软限位余量和整条直线轨迹代价联合排序；同时应
对“固定径向100 mm外推”做长度与方向消融。
